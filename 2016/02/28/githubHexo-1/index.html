怎样在github上利用Hexo框架搭建静态个人博客

发表于 2016-02-28

编者按

作为程序猿，看着别人写的攻略和技术文章长大，都想自己建个博客分享自己的经验。Github大家都在用，作为业界良心，每个账号都分配了免费的个人空间Github Page：，相当于有了免费域名和服务器，我突然明白，从此妈妈再也不用担心我租不起host了。

以前搭网站自己试着造过轮子，深知轮子总是别人造的圆。博客的免费开源框架有很多，但由于github空间上并没有我们自己的数据库，像传统LAMP（Linux+Apache+Mysql+Php）这样的大家伙，当然不适合我们。好在呢有好多静态框架也是萌萌哒，比如githubz自带的jekyii，以及octopress,hugo,还有flask。但作为21世纪新颜控，我想先玩玩Hexo。关于部署Hexo，网上资料很多，但很分散。我这里只是做个整理。

说干就干，搭站什么的最好玩了，至于Hexo和github page的原理，以后几篇再慢慢解释。

1. 本地部署Hexo

先装Hexo，让他在本地localhost先奔跑起来。推荐这个Hexo官网安装手册，这里我就接花献佛啦。
要装零部件有三个：

  1. Node.js （JavaScript解释器，女主的老爹）
  2. git （同步github和本地文件，接送女主的玛莎拉蒂）
  3. Hexo （女主）

1.1 安装 Node.js

安装Node.js的最佳方式是使用nvm，全称Node Version Manager。原因是node.js版本太多，需要nvm来切（调）换（教）。 下面打开Terminal，输入命令行，下载安装nvm，cURL或Wget，任选其一。

1.3 安装Hexo

安装完Node.js以及Git之后，用npm安装Hexo。

1.4 初始化Hexo

本地创建一个名叫Hexo的文件夹。 bash上进到这个目录下，初始化Hexo。 所谓博客框架，hexo能够自动生成一个站点所需的基本组件。

hexo默认IP是0.0.0.0， 端口4000。这时候我们浏览器里访问0.0.0.0：4000，看到hexo的原始外观（默认landscape主题），说明hexo在本地部署成功了。

1.5 换NexT主题

作为程序猿，我怎么能暴露选Hexo就是为了用他的NexT主题呢，太不专业了，么么哒。
NexT用户手册，有你想知道的关于NexT的一切，很详细。这里我列几个关键的步骤。

先克隆NexT主题，

这时候在hexo/themes路径下就多了一个next文件夹。

切换Hexo主题到NexT

打开hexo配置文件hexo/_config.yml,找到theme字段，把landscape改成next。

刷新0.0.0.0：4000，就可以看到一个干净的多的页面。

切换NexT不同外观

NexT通过Scheme提供了三种不同的外观，同样可以通过配置文件切换。打开NexT配置文件hexo/themes/next/_config.yml,找到scheme字段，在Muse，Mist，Pisces里挑一个吧。我更喜欢Mist。
